Definition of Html

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It tells the browser how to display text, images, links, and other elements.

HTML uses "tags" to mark elements like headings, paragraphs, and lists. 


             <h1>This is a Heading</h1>
             <p>This is a paragraph of text.</p>

History of HTML

HTML was created by Tim Berners-Lee in 1991. It started with a simple set of tags and has evolved over time. The latest version is HTML5, which introduced new elements for multimedia, forms, and structure.

  • 1991: First HTML draft
  • 1999: HTML 4.01
  • 2014: HTML5 officially released

Where is HTML used?

HTML is used to build every web page on the internet. It defines the structure of:

  • Websites and blogs
  • Online forms
  • Emails (HTML emails)
  • Web apps (alongside CSS and JavaScript)

Role of HTML in Web Development

HTML is the backbone of web development. It works with CSS (for styling) and JavaScript (for interactivity) to create dynamic, responsive websites.

Understanding HTML is essential for:

  • Creating web pages
  • Building user interfaces
  • Ensuring accessibility
  • Optimizing for search engines (SEO)

                 <!DOCTYPE html>
                 <html>
                 <head>
                 <title>My First Page</title>
                 </head>
                 <body>
                 <h1>Hello, world!</h1>
                 <p>Welcome to my website.</p>
                 </body>
                 </html>

This code creates a simple HTML page with a title and a heading.

Syntax & Structure

Tags and elements

HTML uses tags to define elements on a page. A tag starts with < and ends with >. Most elements have an opening and a closing tag. 


                  <p>This is a paragraph.</p>
                  <h1>Heading 1</h1>

Tags wrap content to describe its role in the document. Together, the tag and its content are called an element.

Attributes

Attributes provide additional information about HTML elements. They are always added inside the opening tag and follow this pattern: name="value". 


                 <img src="image.jpg" alt="A beautiful image">
                 <a href="https://example.com">Visit site</a>
  • href: used in anchor tags to specify link URL
  • src: defines the image source in image tags
  • alt: alternative text for images

Nesting Rules

HTML elements can be nested inside other elements. Proper nesting means opening and closing tags are placed in the correct order. 


                   <div>
                   <p>This is a paragraph inside a div.</p>
                   </div>

Incorrect nesting:


                   <p>This is wrong <div>text</div></p> ❌

Case Sensitivity

HTML is not case-sensitive. That means both <H1> and <h1> are valid. However, the best practice is to always write tags, attributes, and values in lowercase. 


                 <h1>Correct</h1> ✅
                 <H1>Also valid, but not recommended</H1> ⚠️

Elements & Tags

Block vs Inline Elements

Block elements start on a new line and take full width. Inline elements flow within text.


                  <div>This is a block element.</div>
                  <span>This is inline text.</span>

Common Elements

Basic building blocks of HTML pages: headings, paragraphs, and lists.


                  <h1>Heading 1</h1>
                  <p>This is a paragraph.</p>
                  <ul>
                  <li>List item 1</li>
                  <li>List item 2</li>
                  </ul>

Self-closing Tags

Tags without a closing part. Used for images, line breaks, and horizontal rules.


                  <img src="image.jpg" alt="Example image" />
                  <br />
                  <hr />

Semantic Elements

Elements that clearly describe their meaning to browsers and developers.


                      <header>Page header</header>
                      <nav>Navigation links</nav>
                      <article>Article content</article>
                      <footer>Page footer</footer>

Headings & Paragraphs

Headings (h1-h6)

Define the structure and importance of content sections.


                      <h1>Main Title</h1>
                      <h2>Subheading</h2>
                      <h3>Section Title</h3>

Paragraphs

Group related sentences to form blocks of text.


                    <p>This is a paragraph.</p>
                    <p>Another paragraph here.</p>

Text Structure & Semantics

Use semantic tags like <strong> and <em> to add meaning to text.


                     <p>This is  <strong>important</strong> and <em>emphasized</em> text.</p>

Accessibility Tips

  • Use headings in order (h1 to h6) for clear structure.
  • Use semantic elements for better screen reader support.
  • Provide meaningful alternative text for images.

Text & Formatting: Formatting Tags

Bold, Italic, Underline (b, i, u)

Use these tags to style text:

  • <b>: bold text without extra emphasis
  • <i>: italic text for alternate voice or mood
  • <u>: underlined text

                   <p>This is <b>bold</b> text.</p>
                   <p>This is <i>italic</i> text.</p>
                   <p>This is <u>underlined</u> text.</p>

Emphasis Tags (em, strong)

<em> adds emphasis (usually italic), <strong> adds strong importance (usually bold).


                    <p>This is <em>emphasized</em> text.</p>
                    <p>This is <strong>important</strong> text.</p>

Small, Mark, Del, Ins

Useful for specific text effects:

  • <small>: smaller print
  • <mark>: highlighted text
  • <del>: deleted/removed text (strikethrough)
  • <ins>: inserted/added text

                  <p>This is <small>small</small> text.</p>
                  <p>This is <mark>highlighted</mark> text.</p>
                  <p>This was <del>removed</del> text.</p>
                  <p>This is <ins>new</ins> text.</p>

Special Characters & Entities

Use entities to show reserved characters or symbols:


                     &nbsp; (non-breaking space)
                     &amp; (&)
                     &lt; (<)
                     &gt; (>)
                     &copy; (©)

Links & Navigation: Anchor Tags

The <a> Tag

Defines a hyperlink to navigate to another page or location.


                   <a href="https://example.com">Visit Example</a>

Href Attribute

Specifies the URL or path of the link destination.


                    <a href="https://google.com">Google</a>
                    <a href="#section2">Jump to Section 2</a>

Target Attribute

Controls where the link opens:

  • _self (default): opens in the same tab
  • _blank: opens in a new tab/window

                  <a href="https://example.com" target="_blank">Open in new tab</a>

Rel Attribute

Specifies the relationship between the linked resource and the current page.

  • nofollow: tells search engines not to follow the link
  • noopener, noreferrer: improve security when using target="_blank"

                    <a href="https://example.com" target="_blank" rel="noopener noreferrer">Secure Link</a>

Links & Navigation: Navigation Menus

Menu Structure

Navigation menus are lists of links, usually created with <nav> and <ul> elements.


                <nav>
                <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
                </ul>
                </nav>

Dropdown Menus

Dropdowns show submenus when hovering or clicking on a menu item, usually nested lists.


                     <li>Services
                     <ul>
                     <li><a href="#">Design</a></li>
                     <li><a href="#">Development</a></li>
                     </ul>
                     </li>

Responsive Navigation

Responsive menus adapt for different screen sizes, often with hamburger buttons and toggles.

Example: CSS media queries and JavaScript toggle the menu visibility on small screens.

ARIA Roles for Nav

Use ARIA roles to improve accessibility:

  • role="navigation": identifies the navigation section
  • aria-label="Main menu": describes the navigation purpose

                    <nav role="navigation" aria-label="Main menu">
                      ...
                   </nav>

Forms: Form Basics

The <form> Tag

Wraps form controls to submit user input to a server.


                     <form action="/submit" method="post">
                       ...
                     </form>

Method Attribute (GET, POST)

Defines how data is sent:

  • GET: appends data in URL (visible)
  • POST: sends data in request body (secure)

                    <form method="get"> or <form method="post">

Action Attribute

Specifies the URL where form data is sent on submission.


                   <form action="/process-data"> ... </form>

Basic Form Structure

Typical form includes labels and inputs:


                  <form>
                  <label for="name">Name:</label>
                  <input type="text" id="name" name="name">
                  <button type="submit">Submit</button>
                  </form>

Forms: Input Types

Text, Email, Password

Common input fields:


                    <input type="text" name="username" placeholder="Username">
                    <input type="email" name="email" placeholder="Email">
                    <input type="password" name="password" placeholder="Password">

Radio & Checkbox

Used for single or multiple selections:


                   <input type="radio" name="gender" value="male"> Male
                   <input type="radio" name="gender" value="female"> Female

                   <input type="checkbox" name="subscribe"> Subscribe to newsletter

Date, File, Range

Specialized inputs:


                  <input type="date" name="dob">
                  <input type="file" name="upload">
                  <input type="range" name="volume" min="0" max="100">

Disabled & Readonly

Control input usability:

  • disabled: input is unusable and excluded from form submission
  • readonly: input is view-only but included in submission

                   <input type="text" disabled>
                  <input type="text" readonly value="Cannot edit">

Forms: Validation

Required & Pattern Attributes

required forces users to fill an input before submitting.

pattern uses regex to validate input format.


                   <input type="text" required>
                   <input type="text" pattern="[A-Za-z]{3,}" title="At least 3 letters">

Min & Max Values

Set minimum and maximum limits for numeric or date inputs.


                   <input type="number" min="1" max="10">
                   <input type="date" min="2023-01-01" max="2025-12-31">

Custom JavaScript Validation

Use JS to add custom checks beyond HTML validation.


                  const form = document.querySelector('form');
                  form.addEventListener('submit', e => {
                  const input = form.elements['username'];
                  if(input.value.length < 3) {
                  e.preventDefault();
                  alert('Username must be at least 3 characters');
                    }
                 });

Validation Examples

Combining HTML and JS validation for better UX:


                    <input type="email" required pattern=".+@.+\.com">
                    <script>
                    // Additional JS checks here
                   </script>

Semantic HTML

What is Semantic HTML?

HTML that uses meaningful tags to describe content clearly.

Common Semantic Tags

Examples include:

  • <header>
  • <nav>
  • <article>
  • <section>
  • <footer>

ARIA & Accessibility

ARIA roles enhance semantic meaning for assistive tech.


                  <nav role="navigation"> ... </nav>
                  <main role="main"> ... </main>

Benefits of Semantic HTML

  • Improves accessibility
  • Easier maintenance
  • Better SEO

SEO Advantages

Search engines understand page structure better with semantic tags.

Best Practices

Clean HTML Structure

Write well-organized, properly nested code for readability and performance.

Consistent Naming Conventions

Use clear, consistent class and ID names to simplify development and maintenance.

Responsive Design Basics

Design layouts that adapt to different screen sizes using CSS media queries.

Accessibility Tips

  • Use semantic HTML
  • Provide alt text for images
  • Use ARIA roles where necessary

Avoid Inline Styles

Use external CSS instead of inline styles to keep code clean and maintainable.
